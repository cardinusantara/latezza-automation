# Meta Ads Report Automation & WhatsApp Gateway

This repository contains a unified, high-performance automated Meta Ads reporting system. Every day at **09:00 AM (Asia/Jakarta)**, the system automatically fetches Meta Ads performance metrics, uses Gemini AI to generate copywriting summaries and qualitative insights, compiles a dark-themed HTML report, and broadcasts the report link and summary to a designated WhatsApp Group JID.

---

## 📂 Project Structure

The project is structured into two self-contained, modular directories:

```
automation/
├── .env                       # Global environment configuration (shared)
├── .gitignore
├── ads-analysis/              # Ads metrics fetching, processing & HTML report compilation
│   ├── automation.js          # Core analysis script (executes as child process)
│   ├── template.html          # HTML dashboard template
│   ├── report.html            # Compiled report (generated by script)
│   ├── package.json           # Specific dependencies for ads analysis (Gemini SDK)
│   └── node_modules/          
└── whatsapp-gateway/          # Fastify server, Baileys WA connection & node-cron scheduler
    ├── gateway.js             # persistent Fastify server & cron job
    ├── package.json           # Specific dependencies for Baileys/Fastify/node-cron
    ├── whatsapp-session/      # Persistent authenticated WhatsApp session
    └── node_modules/
```

## ⚙️ Environment Configuration (`.env`)

Create a `.env` file in the root directory with the following variables:

```env
# Meta Ads API Credentials
META_ACCESS_TOKEN=your_meta_graph_api_token
META_AD_ACCOUNT_ID=1319452786362265

# Google Gemini API Key
GEMINI_API_KEY=your_gemini_api_key

# WhatsApp Broadcast Settings
WHATSAPP_GROUP_JID=120363427625298309@g.us
PUBLIC_REPORT_URL=https://brock-concussive-unsuggestively.ngrok-free.dev
```

> [!IMPORTANT]
> **Meta API Credentials**: The Access Token requires `ads_read` and `business_management` permissions. Ensure it is a long-lived user token or a System User token from your Facebook Business Manager to prevent daily expiration.

## 🚀 Setup & Execution

### 1. Install Dependencies
Run `npm install` inside both module directories:
```bash
# Install Ads Analysis dependencies
cd ads-analysis
npm install

# Install WhatsApp Gateway dependencies
cd ../whatsapp-gateway
npm install
```

### 2. Start the WhatsApp Gateway
Launch the gateway from the `whatsapp-gateway` directory:
```bash
node gateway.js
```
- **First-time Authentication**: A QR code will print in the terminal. Scan it using your WhatsApp account (linked devices) to authenticate.
- **Persistent State**: The session credentials will be saved in `whatsapp-session/` so you don't need to re-authenticate on restart.
- **Cron Active**: On startup, Fastify arms the `node-cron` job to run every morning at 09:00 AM (Asia/Jakarta).

### 3. Setup Tunnel (ngrok)
To allow people outside localhost to view the dashboard report, tunnel port `3001` via ngrok:
```bash
ngrok http 3001
```
Then copy the public HTTPS domain (e.g. `https://brock-concussive-unsuggestively.ngrok-free.dev`) and set it as `PUBLIC_REPORT_URL` in your `.env`.

## 📡 API Reference (Port 3001)

The gateway exposes the following HTTP endpoints:

### 1. Trigger Report Run (`POST /trigger-analysis`)
Runs the ads analysis immediately in the background and broadcasts the message to WhatsApp.
- **Request Headers**: `Content-Type: application/json`
- **Request Body**: `{}`
- **Response**:
  ```json
  {
    "status": "success",
    "message": "Analysis triggered in background."
  }
  ```

### 2. View Report Dashboard (`GET /report-html`)
Serves the dynamic compiled `report.html` page directly to the browser with standard HTML rendering (`Content-Type: text/html`).

### 3. List WhatsApp Groups (`GET /groups`)
Returns a list of all participating WhatsApp group names and JIDs from the connected account. Use this to find target group IDs.
- **Response**:
  ```json
  [
    { "jid": "120363427625298309@g.us", "name": "Testing-group" }
  ]
  ```

### 4. Direct WhatsApp Message (`POST /send-message`)
Sends a custom text message to any phone or group JID.
- **Request Body**:
  ```json
  {
    "jid": "120363427625298309@g.us",
    "text": "Your message text here..."
  }
  ```
